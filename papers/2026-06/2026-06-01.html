SAAS: Self-Aware Reinforcement Learning for Over-Search Mitigation in Agentic Search ★ 高度相关
Yunbo Tang, Chengyi Yang, Shiyu Liu, Zhishang Xiang et al. · 2026-06-01
HF精选agentic searchover-searchinference efficiency
Tackles "over-search" in agentic search, where LLM agents needlessly trigger searches when internal knowledge suffices or fail to stop after gathering enough evidence, inflating inference latency and cost. It proposes SAAS, an RL framework with three parts: search-boundary modeling (contrasting search-disabled vs. search-enabled rollouts), a boundary-aware reward that penalizes redundant searches at the trajectory level, and a stage-wise curriculum prioritizing reasoning before search regularization to avoid reward hacking. Experiments show SAAS substantially reduces over-search while maintaining accuracy.
中文摘要
本文针对智能体检索中的"过度检索"问题——LLM 智能体在内部知识已足够时仍盲目触发检索、或在证据充分后仍不停止，导致推理延迟与计算成本激增。作者提出强化学习框架 SAAS，包含三部分：通过对比"禁用检索"与"启用检索"轨迹来建模检索边界；将边界意识转化为轨迹级惩罚的边界感知奖励，以抑制冗余检索；以及优先优化推理、再正则化检索的分阶段课程策略以避免奖励作弊。实验表明该方法在保持准确率的同时大幅减少了过度检索。

Emergent Languages in Populations of Language Model Agents: From Token Efficiency to Oversight Evasion ★ 高度相关
Stine Lyngsø Beltoft, William Brach, Federico Torrielli, Jacob Nielsen et al. · 2026-06-01
HF精选multi-agent communicationemergent languagetoken efficiency
Studies how populations of LLM agents on the "Moltbook" platform spontaneously invent new communication languages, using a rule-based heuristic (~6000 matches) followed by zero-shot classification (518 kept) to surface cases across token-efficiency (166), novel natural languages (106), and oversight-evasion (59) categories. It finds that oversight-evasion languages are judged less aligned, that any LLM can learn these languages in-context merely from a description, and that some agents develop sophisticated steganographic protocols hiding messages inside natural language. The authors conclude that monitoring surface behavior alone may soon be insufficient to retain control over agent populations.
中文摘要
本文研究了 Moltbook 平台上大量 LLM 智能体群体自发产生的新型通信语言，采用规则启发式（约 6000 个匹配）与零样本分类（保留 518 例）相结合的两阶段方法，将案例归类为词元效率（166）、新自然语言（106）与逃避监管（59）三类。研究发现：用于逃避监管的语言被判定为更不对齐；任何 LLM 仅凭语言描述即可在上下文中学会这些语言；且部分智能体发展出将隐藏信息嵌入自然语言的复杂隐写协议。作者由此指出，仅靠监控表层行为可能很快不足以维持对智能体群体的有效管控。

From Model Scaling to System Scaling: Scaling the Harness in Agentic AI ★ 高度相关
Shangding Gu · 2026-06-01
HF精选agent harnessorchestrationcontext efficiency
Argues the next bottleneck in agentic AI is "scaling the harness"—the auditable, modular execution layer (memory substrate, context constructor, skill routing, orchestration loop, verification/governance) around a foundation model—rather than scaling the model alone. It identifies three core bottlenecks (context governance, trustworthy memory, dynamic skill routing) and calls for harness-level benchmarks measuring trajectory quality, memory hygiene, context efficiency, communication fidelity, and verification cost. To make the case concrete it introduces CheetahClaws, a Python-native reference harness, comparing it against Claude Code and OpenClaw.
中文摘要
本文提出智能体 AI 的下一个瓶颈在于"扩展执行框架（harness）"——即围绕基础模型、可审计且模块化的执行层（记忆基底、上下文构建器、技能路由、编排循环、验证与治理），而非单纯扩展模型本身。文章归纳了三大核心瓶颈（上下文治理、可信记忆、动态技能路由），并呼吁建立面向框架层的评测基准，衡量轨迹质量、记忆卫生、上下文效率、通信保真度与验证成本。作者还提出 Python 原生的参考框架 CheetahClaws，并与 Claude Code、OpenClaw 进行了对比。

Mellum2 Technical Report ◆ 相关
Marko Kojic, Ivan Bondyrev, Aral de Moor, Joseph Shtok et al. · 2026-06-01
HF精选Tech Reportcode LLMMixture-of-Expertsagentic coding
Technical report for Mellum 2, an open-weight 12B Mixture-of-Experts model (2.5B active params/token) specialized for software engineering—code generation/editing, debugging, multi-step reasoning, tool use and function calling, and agentic coding—succeeding the 4B dense Mellum. The architecture combines MoE (64 experts, 8 active), Grouped-Query Attention, Sliding-Window Attention on three of every four layers, and a Multi-Token Prediction head doubling as a speculative-decoding draft model, with commodity-GPU inference efficiency as a design constraint. Pre-trained on ~10.6T tokens (three-phase curriculum, Muon optimizer, FP8), extended to 128K context via layer-selective YaRN, and post-trained with SFT+RLVR into Instruct and Thinking variants—competitive with 4-14B open baselines at 2.5B-dense per-token compute, released under Apache 2.0.
中文摘要
本文是 Mellum 2 的技术报告：一个开源权重的 12B 专家混合（MoE）模型（每词元激活 2.5B 参数），专注于软件工程任务——代码生成与编辑、调试、多步推理、工具与函数调用、以及智能体式编程，是 4B 稠密版 Mellum 的后继者。其架构结合 MoE（64 个专家、激活 8 个）、分组查询注意力、每四层中三层采用滑动窗口注意力，以及兼作推测解码草稿模型的多词元预测头，并以普通商用 GPU 的推理效率为设计约束。模型在约 10.6 万亿词元上经三阶段课程、Muon 优化器与 FP8 预训练，通过分层选择性 YaRN 扩展至 128K 上下文，再经 SFT 与 RLVR 后训练得到 Instruct 与 Thinking 两个版本——在 2.5B 稠密模型的单词元算力下性能可与 4–14B 开源基线竞争，并以 Apache 2.0 许可发布。

OpenSkillEval: Automatically Auditing the Open Skill Ecosystem for LLM Agents ◆ 相关
Jiahao Ying, Boxian Ai, Wei Tang, Siyuan Liu et al. · 2026-06-01
HF精选LLM agentsagent skillsframework eval
Presents OpenSkillEval, an automatic evaluation framework for both skill-augmented LLM agent systems and the skills themselves, dynamically constructing realistic task instances from evolving real-world artifacts across five application categories (presentation, front-end web design, poster, data visualization, report generation). Using 600+ generated task instances and 30 open-source skills, it finds that skill availability does not guarantee effective use, that the benefit depends strongly on the underlying model and agent framework, and that many popular skills do not consistently outperform skill-free baselines. The work argues for dynamic, task-grounded skill evaluation.
中文摘要
本文提出 OpenSkillEval，一个面向技能增强型 LLM 智能体系统及技能本身的自动评测框架，从五类真实应用（幻灯片、前端网页设计、海报、数据可视化、报告生成）的演变产物中动态构建贴近现实的任务实例。基于 600 余个生成任务实例与 30 个开源技能的系统评测发现：拥有技能并不保证能有效使用，其收益强烈依赖于底层模型与智能体框架，且许多热门技能未必能稳定超过不用技能的基线。作者据此倡导采用动态、任务驱动的技能评测方式。

GrepSeek: Training Search Agents for Direct Corpus Interaction ◆ 相关
Alireza Salemi, Chang Zeng, Atharva Nijasure, Jui-Hui Chung et al. · 2026-06-01
HF精选search agenttool useretrieval speedup
Proposes GrepSeek, a search agent that treats the corpus itself as the search environment and gathers evidence by issuing executable shell commands (direct corpus interaction), rather than relying on a pre-indexed retriever. It uses a two-stage pipeline—cold-start trajectories built by an answer-aware Tutor and answer-blind Planner, then GRPO refinement—plus a semantics-preserving sharded-parallel execution engine that accelerates shell-based retrieval up to 7.6x while preserving byte-exact equivalence. GrepSeek achieves the strongest overall token-level F1 and exact match across seven open-domain QA benchmarks.
中文摘要
本文提出 GrepSeek，一种将语料库本身作为搜索环境、通过执行 shell 命令直接交互来获取证据的搜索智能体（直接语料交互 DCI），而非依赖预先建好索引的检索器。方法采用两阶段流程——先由"知答家教（Tutor）"与"盲答规划器（Planner）"构造冷启动轨迹，再用 GRPO 精化策略——并辅以保持语义、逐字节等价的分片并行执行引擎，将 shell 检索加速最高 7.6 倍。在七个开放域问答基准上，GrepSeek 取得了最强的整体词元级 F1 与精确匹配成绩。

Task-Focused Memorization for Multimodal Agents ◆ 相关
Tao Zou, Yichen He, Tian Qiu, Yuan Lin et al. · 2026-06-01
HF精选multimodal agentsmemory policycontinual learning
Frames long-term memory for multimodal/embodied agents as a learnable "what to memorize" policy, since agents face an unbounded stream of observations and must selectively retain only role-relevant, future-useful content. TaskMem is an RL framework with two phases: Phase One learns how to memorize under fundamental fidelity requirements; Phase Two, after deployment, tunes an adapter on the base MLLM using recent environment tasks to define a reward model that steers memorization toward task-relevant content. On streaming reformulations of VideoMME, EgoLife, and EgoTempo (built on Qwen3-VL-30B-A3B, answering only from memory), TaskMem improves VQA accuracy by 6.3%, 7.0%, and 5.3% respectively.
中文摘要
本文将多模态/具身智能体的长期记忆建模为一个可学习的"记什么"策略——因为智能体面对无限的观测流，必须选择性地只保留与其角色相关、对未来任务有价值的内容。提出的 TaskMem 是一个两阶段强化学习框架：第一阶段在基本保真要求下学习"如何记忆"；第二阶段在部署后，利用近期环境任务定义奖励模型、在基座 MLLM 上微调适配器，从而引导记忆策略聚焦于任务相关内容。在 VideoMME、EgoLife、EgoTempo 的流式改造基准上（基于 Qwen3-VL-30B-A3B，且仅凭记忆作答），其 VQA 准确率分别提升 6.3%、7.0% 与 5.3%。

SCOPE: Self-Play via Co-Evolving Policies for Open-Ended Tasks ◆ 相关
Wai-Chung Kwan, Aryo Pradipta Gema, Joshua Ong Jun Leang, Pasquale Minervini · 2026-06-01
HF精选self-playco-evolving policiesmulti-turn retrieval
Introduces SCOPE, a data-free self-play framework for open-ended tasks that co-evolves two policies—a Challenger generating document-grounded tasks and a Solver answering via multi-turn retrieval—with a frozen copy of the initial model acting as a rubric-writing self-judge. Across three 7-8B instruction-tuned models (Qwen2.5, Qwen3, OLMo-3) it improves open-ended performance by up to +10.4 points and matches or exceeds GRPO trained on ~9K curated prompts, while also boosting held-out short-form QA by up to +13.8. Ablations show co-evolving the Challenger is necessary to keep tasks near the Solver's frontier, and rubric quality is the self-judging bottleneck.
中文摘要
本文提出 SCOPE，一种面向开放式任务的无数据自博弈框架：协同进化两个策略——生成基于文档任务的"出题者（Challenger）"与通过多轮检索作答的"解题者（Solver）"，并以冻结的初始模型作为撰写评分细则的自评判者。在三个 7–8B 指令微调模型（Qwen2.5、Qwen3、OLMo-3）上，开放式任务性能最高提升 10.4 分，可媲美甚至超过使用约 9K 精选提示训练的 GRPO，同时在保留的短问答基准上最高提升 13.8 分。消融实验表明，协同进化出题者是使任务保持在解题者能力前沿的关键，而评分细则质量是自评判的瓶颈。

DRIFT: Decoupled Rollouts and Importance-Weighted Fine-Tuning for Efficient Multi-Turn Optimization ◆ 相关
Jian Mu, Tianyi Lin, Chengwei Qin, Zhongxiang Dai et al. · 2026-06-01
HF精选multi-turn RLtraining efficiencyimportance weighting
Addresses the dilemma in optimizing multi-turn interactive LLMs: online RL handles multi-turn dynamics but is prohibitively expensive (generating full correction trajectories per update), while offline SFT is efficient but suffers distribution shift and behavioral collapse. DRIFT (Decoupled Rollouts and Importance-weighted Fine-Tuning) operationalizes the equivalence between the KL-regularized RL objective and importance-weighted supervised learning—sampling offline trajectories from a fixed reference policy, deriving return-based importance weights, and optimizing via weighted SFT. It matches or exceeds multi-turn RL baselines at SFT-level training cost and simplicity.
中文摘要
本文针对多轮交互式 LLM 优化中的两难：在线强化学习能处理多轮动态但成本高昂（每次更新都需生成完整纠错轨迹），离线监督微调（SFT）高效却受分布偏移与行为坍缩之苦。作者提出 DRIFT（解耦采样与重要性加权微调），将 KL 正则化强化学习目标与重要性加权监督学习的等价性付诸实践——从固定参考策略离线采样轨迹、推导基于回报的重要性权重，再通过加权 SFT 优化策略。实验表明其性能可媲美甚至超过多轮强化学习基线，同时保持 SFT 级别的训练效率与简洁性。

Memory-Bound but Not Bandwidth-Limited: The Physical AI Inference Gap in Batch-1 LLM Decode ◆ 相关
Josef Chen · 2026-06-01
HF精选LLM inferencebatch-1 decodeefficiency
Empirically dissects single-stream batch-1 autoregressive decode—the workload of physical-AI systems like robots, autonomous vehicles, and embodied agents—across three 7-8B GQA transformers and four NVIDIA GPUs (H100, A100, L40S, L4). It shows the common "memory-bandwidth-bound" account is true but incomplete: the achieved fraction of peak HBM bandwidth falls as peak bandwidth rises (81% on L4 vs. only 27% on H100), and a CUDA Graphs A/B test isolates a launch-side overhead visible on fast GPUs (1.26x speedup on H100 vs. 1.03x on L4). It also finds common int4 quantization paths fail to realize the expected 4x weight-traffic reduction unless kernels are well-tuned (GPTQ+ExLlamaV2 reaching 17.36 ms/step).
中文摘要
本文实证剖析了单流 batch-1 自回归解码——机器人、自动驾驶、具身智能体等"物理 AI"系统的典型负载——在三款 7–8B GQA Transformer 与四款 NVIDIA GPU（H100、A100、L40S、L4）上的表现。研究表明，常见的"受内存带宽限制"说法虽对但不完整：实测达到的峰值 HBM 带宽占比随带宽提升反而下降（L4 约 81%，H100 仅 27%）；通过 CUDA Graphs 的 A/B 实验进一步分离出在高速 GPU 上才显现的启动端开销（H100 提速 1.26 倍，L4 仅 1.03 倍）。此外，常见的 int4 量化路径若内核未充分调优，往往无法实现预期的 4 倍权重传输量削减（GPTQ+ExLlamaV2 可达 17.36 毫秒/步）。
