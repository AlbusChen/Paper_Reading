When Does Multi-Agent RL Improve LLM Workflows? Workflow, Scale, and Policy-Sharing Tradeoffs ★ 高度相关
· 2026-06-02
multi-agent
This paper studies when end-to-end RL training of multi-agent LLM workflows actually beats the base models, contrasting Shared-Policy training (all roles update one policy) with Isolated-Policy training (each role keeps its own parameters) across three workflow topologies, math and code tasks, and three model scales (0.6B-4B). It finds multi-agent RL usually helps, but the gains depend jointly on workflow, task, and scale rather than on policy sharing alone: Isolated-Policy reaches higher peak accuracy yet more often falls off a 'terminal accuracy cliff,' while Shared-Policy merely redistributes failure into different patterns. The authors trace these regimes to role-level gradient dynamics induced by workflow topology and policy routing-e.g., parallel same-role agents amplifying per-role gradients, and the shared policy being captured by a dominant role.
中文摘要
本文研究端到端强化学习训练多智能体LLM工作流何时真正优于基座模型，系统对比了"共享策略"（所有角色共用一套参数）与"独立策略"（每个角色独立参数）两种范式，覆盖三种工作流拓扑、数学与代码任务以及0.6B至4B三种模型规模。研究发现多智能体RL通常能带来提升，但增益取决于工作流、任务与规模的联合作用，而非策略共享与否：独立策略峰值精度更高却更易跌入"终端精度悬崖"，共享策略则只是把失败重新分布为不同形态。作者进一步用工作流拓扑与策略路由诱导的角色级梯度动力学解释了这些现象（如并行同角色智能体放大单角色梯度、共享策略被主导角色"俘获"等）。

Multi-Agent Computer Use ★ 高度相关
· 2026-06-02
multi-agent
Arguing that single serial computer-use agents (CUAs) are suboptimal for complex long-horizon tasks, this paper proposes multi-agent computer use (MACU): a manager model decomposes a task into a directed acyclic graph (DAG) of subgoals, dispatches parallel CUA subagents on the DAG's ready frontier, and continuously revises the DAG (adding, canceling, rewriting nodes) as findings arrive, forwarding observations that downstream agents cannot re-observe. MACU improves over strong single-agent baselines by 3.4-25.5% on OSWorld and several web-navigation benchmarks and shows more favorable test-time scaling. Notably for efficiency, it cuts average wall-clock completion time by about 1.5x on the long-horizon Odysseys benchmark while solving tasks where single agents get stuck.
中文摘要
本文指出当前以单个串行智能体部署的计算机使用智能体（CUA）难以胜任复杂长程任务，提出"多智能体计算机使用"（MACU）：由管理者模型将任务分解为有向无环图（DAG），在DAG就绪前沿上并行派发CUA子智能体，并随子智能体返回的新信息持续修订DAG（增删与改写节点），同时把下游难以重新观测的信息通过管理者与DAG结构向前传递。MACU在OSWorld桌面及多个网页导航基准上较强单智能体基线提升3.4%–25.5%，并展现出更优的测试时扩展性。在效率方面尤为突出：它在长程基准Odysseys上将平均完成耗时缩短约1.5倍，并能解决单智能体会卡住的任务。

Crafter: A Multi-Agent Harness for Editable Scientific Figure Generation from Diverse Inputs ◆ 相关
· 2026-06-02
multi-agent
Publication-quality scientific figures are labor-intensive, yet existing generators each target a single figure type from text-only input and emit non-editable raster output. The authors argue that such structured, component-based layouts need a coordinating 'harness' rather than a stronger backbone, and instantiate Crafter, a multi-agent harness that generalizes across figure types and input conditions without architectural changes (plus CraftEditor for raster-to-editable-SVG conversion and the CraftBench benchmark). Crafter substantially outperforms standalone generators and an agentic baseline, with ablations confirming each agent component contributes independently.
中文摘要
出版级科研配图制作耗时费力，而现有生成系统通常仅针对单一图表类型、仅支持纯文本输入且输出为不可局部编辑的栅格图。作者认为此类由离散语义组件构成的结构化版面所需的是协同框架（harness）而非更强的骨干模型，据此提出多智能体协同框架Crafter，无需改动架构即可泛化到多种图表类型与输入条件（并配套CraftEditor实现栅格图到可编辑SVG的转换及CraftBench基准）。实验表明Crafter显著优于独立生成器与智能体基线，消融实验证实各智能体组件均有独立贡献。

SkillAdaptor: Self-Adapting Skills for LLM Agents from Trajectories ◆ 相关
· 2026-06-02
LLM agents increasingly rely on reusable external skills for long-horizon tasks, but existing training-free skill-adaptation pipelines update from full trajectories or session-level feedback, giving coarse failure attribution and unstable, overly broad revisions. SkillAdaptor is a training-free, step-level skill-adaptation framework with explicit failure attribution: given a failed trajectory it identifies the first actionable fault step, links responsibility to candidate skills, and applies targeted updates under explicit acceptance checks while keeping the backbone frozen. Across WebShop, PinchBench, and Claw-Eval with several frontier models it consistently beats no-skill and prior skill-adaptation baselines (e.g., +1.7 on WebShop success rate).
中文摘要
LLM智能体在长程交互任务中越来越依赖可复用的外部技能，但现有免训练技能适配流程多基于完整轨迹或会话级反馈进行更新，导致失败归因粗糙、修订不稳定且过于宽泛。SkillAdaptor提出一种带显式失败归因的免训练步级技能适配框架：给定失败轨迹，先定位首个可纠正的错误步，将责任归因到候选技能，再在显式验收检查下进行有针对性的更新，同时冻结骨干模型。在WebShop、PinchBench与Claw-Eval三个基准（搭配多种前沿模型）上，它一致优于无技能与已有技能适配基线（如WebShop成功率+1.7）。

MCP-Persona: Benchmarking LLM Agents on Real-World Personal Applications via Environment Simulation ◆ 相关
· 2026-06-02
The Model Context Protocol (MCP) has become a standard for connecting LLMs to external tools, yet existing benchmarks emphasize generic information-seeking and overlook personalized tools that interact with individual accounts or local databases. MCP-Persona is the first benchmark targeting real-world personalized MCP tools, spanning social platforms (Reddit, Rednote) and enterprise collaboration suites (Lark/Feishu, Slack). Extensive experiments show that state-of-the-art agents struggle significantly with personalized tool use, exposing a practical gap that current evaluations miss.
中文摘要
模型上下文协议（MCP）已成为连接大模型与外部工具的重要标准并在个人应用中快速普及，但现有基准多聚焦通用信息检索，忽视了涉及个人账户或本地数据库的个性化工具。本文提出MCP-Persona——首个面向真实个性化MCP工具的基准，覆盖社交平台（Reddit、小红书）与企业协作套件（飞书Lark、Slack）。大量实验表明，当前最先进的智能体在个性化工具使用上明显吃力，揭示了现有评测所忽略的实际能力缺口。

Masking Stale Observations Helps Search Agents -- Until It Doesn't: A Regime Map and Its Mechanism ◆ 相关
· 2026-06-02
Long-horizon search agents accumulate large amounts of retrieved content across many tool calls, making context-budget efficiency critical; a minimal intervention is to mask stale observations, but when this helps and why has been unclear. Sweeping backbones from 4B to 284B parameters and three retrievers on offline and live-web benchmarks, the authors find the accuracy gain from masking follows an asymmetric inverted-U against the model's no-masking accuracy-a plateau under weak retrievers, a peak when a strong retriever meets a mid-capacity model, and a sharp collapse once the model saturates. Mechanistically, masking is a token-for-turn trade-off: it helps only when the extra turns convert failures into successes, and hurts when it removes evidence the model would otherwise have used.
中文摘要
长程检索智能体会在多次工具调用中累积大量检索内容，使上下文预算效率变得关键；一个最小化干预是随轨迹推进屏蔽过时观测，但其何时有效、为何有效一直不清晰。作者在4B至284B参数的多种骨干模型、三种检索器以及离线与实时网页基准上系统扫描，发现屏蔽带来的精度增益相对于"无上下文管理时的精度"呈非对称倒U形：弱检索器下趋于平台、强检索器配中等能力模型时达到峰值、模型能力饱和后急剧崩塌。机理上，屏蔽是一种"以轮次换token"的权衡：仅当额外轮次能把失败转为成功时才有益，而当它删去模型本可利用的证据时则有害。
