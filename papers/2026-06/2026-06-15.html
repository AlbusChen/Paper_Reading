How Task Structure Limits Multi-Agent Success: An Information-Theoretic Analysis ★ 高度相关
Shi Pan, Ming Luo · 2026-06-11
cs.ITcs.LGsingle-vs-multicommunication costtask heterogeneityinter-agent communicationmulti-agentsingle-agentagent communication
This paper gives an information-theoretic account of when multi-agent systems fail to beat single-agent systems: bounded inter-agent communication creates a bottleneck determined by the task constraint graph. The authors define a minimum cut cost and show success probability can decay exponentially when task constraints are tightly coupled, with validation on synthetic tasks and SWE-bench data. The practical takeaway is that high-coupling tasks may need restructuring rather than simply adding agents or more messages.
中文摘要
本文从信息论角度解释多智能体为何不一定优于单智能体：当任务约束图被切分且跨智能体通信受限时，会出现由最小割成本刻画的信息瓶颈。作者证明在高耦合任务中成功率可能随瓶颈指数下降，并在合成实验和 SWE-bench 数据上验证。结论是，高耦合任务应优先重构任务分解，而不是单纯增加智能体或通信轮数。

CoAgent: Concurrency Control for Multi-Agent Systems ★ 高度相关
Hongtao Lyu, Dingyan Zhang, Mingyu Wu, Xingda Wei et al. · 2026-06-13
cs.DCcs.AIcs.MAshared memoryorchestrationconcurrency controlmulti-agent coordinationmulti-agentsingle agent
CoAgent addresses a concrete coordination problem in parallel LLM-agent workflows: multiple agents mutating the same repository, cluster, or document. Its MTPO protocol fixes a serialization order, lets agents read order-filtered state, applies writes speculatively, and asks affected agents to repair plans using LLM judgment and undoable tools. Experiments show near-serial correctness with meaningful speed and cost gains, making shared-state coordination a first-class multi-agent systems issue.
中文摘要
CoAgent 研究并行 LLM 智能体同时修改同一代码库、集群或文档时的共享状态冲突问题。其 MTPO 协议预先固定串行顺序，提供按顺序过滤的读视图，允许写入先投机执行，并通过通知受影响智能体重新判断和修补计划来恢复一致性。实验显示该方法接近串行正确性，同时保留并发带来的时间与成本收益，凸显共享状态协调是多智能体系统的核心问题。

Security Engineering of OpenClaw: Analyzing Attack Surface Expansion and Trust-Boundary Violations ★ 高度相关
Saeid Jamshidi, Arghavan Moradi Dakhel, Kawser Wazed Nafi, Foutse Khomh · 2026-06-12
cs.CRsingle-vs-multimulti-agent securitytrust boundaryorchestrationmulti-agentsingle agent
This paper studies how moving from one action-capable LLM agent to a seven-agent OpenClaw setup expands the security attack surface. With any-agent action execution, compromise probability rises sharply, prompt injection propagates across agents, and privilege drift increases even though the underlying models are unchanged. The result is a strong cautionary single-vs-multi finding: output aggregation and trust-boundary design can dominate model-level safety.
中文摘要
本文分析从单个可执行动作的 LLM 智能体扩展到七智能体 OpenClaw 系统后，安全攻击面如何扩大。在“任一智能体可触发执行”的设置下，妥协概率显著上升，提示注入会跨智能体传播，权限漂移也会增加，而底层模型并未改变。该结果提醒，多智能体的输出聚合和信任边界设计可能比单个模型安全性更决定系统风险。

cs.AIcs.CLcs.MAsingle-vs-multicost efficiencytask decompositionagent architecturemulti-agentsingle-agent
This paper directly challenges the assumed advantage of multi-agent systems by comparing automatically generated MAS against a strong single-agent Chain-of-Thought with Self-Consistency baseline. Across standard reasoning and interactive workflows, generated MAS underperform while costing up to 10x more; a diagnostic dataset shows expert-designed MAS can work when decomposition, context separation, and parallelism are real. The main claim is that current automated MAS often add architectural complexity without functional multi-agent utility.
中文摘要
