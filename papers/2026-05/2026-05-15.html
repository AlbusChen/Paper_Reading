Beyond Individual Intelligence: Surveying Collaboration, Failure Attribution, and Self-Evolution in LLM-based Multi-Agent Systems ★ 高度相关
· 2026-05-15
multi-agent
LLM-based multi-agent systems amplify error propagation across agents and rounds, making failures hard to diagnose. This survey organizes the field around the LIFE progression — Lay capability foundation, Integrate agents through collaboration, Find faults through attribution, Evolve via self-improvement — and formalizes the causal dependencies between stages. It identifies research gaps at stage boundaries toward closed-loop, self-organizing collective intelligence.
中文摘要
本文是一篇面向大模型多智能体系统的综述，指出多智能体协作会放大跨智能体、跨轮次的错误传播，使故障难以定位。作者提出 LIFE 框架（能力奠基—协作整合—故障归因—自主演化）统一组织该领域，并形式化各阶段间的因果依赖。文章重点剖析阶段衔接处的研究空白，指向具备闭环故障诊断与自我结构优化的集体智能。

Learning to Communicate Locally for Large-Scale Multi-Agent Pathfinding ★ 高度相关
· 2026-05-15
multi-agent
Multi-agent pathfinding (MAPF) is NP-hard, and existing decentralized learning-based solvers lack effective coordination and struggle to scale once communication is added. The paper introduces LC-MAPF, a learnable local communication module enabling multi-round information exchange between neighboring agents via efficient feature sharing. It outperforms prior IL- and RL-based MAPF solvers across metrics while preserving scalability, easing the usual communication–scalability tradeoff.
中文摘要
多智能体路径规划（MAPF）是 NP 难问题，现有基于学习的去中心化方法缺乏有效协调机制，且引入通信后难以扩展。本文提出 LC-MAPF，设计可学习的局部通信模块，使相邻智能体通过高效特征共享进行多轮信息交换。该方法在多项指标上优于已有的模仿学习与强化学习方案，同时不损失可扩展性，缓解了通信与可扩展性之间的权衡。

Achieving Gold-Medal-Level Olympiad Reasoning via Simple and Unified Scaling · 低相关
· 2026-05-15
Tech Report
This 77-page technical report presents a simple, unified recipe for turning a post-trained reasoning backbone into an olympiad-level solver: a reverse-perplexity SFT curriculum to instill rigorous proof-search and self-checking, a two-stage RL pipeline (verifiable then proof-level rewards), and test-time scaling. The resulting 30B-A3B model SU-01, trained on ~340K trajectories plus 200 RL steps, sustains 100K+ token reasoning and reaches gold-medal level on IMO 2025/USAMO 2026 and IPhO 2024/2025, generalizing beyond math and physics.
中文摘要
这是一份 77 页的技术报告，提出将后训练推理骨干转化为奥赛级求解器的统一方法：先用逆困惑度课程进行 SFT 以注入严谨的证明搜索与自检行为，再经“可验证奖励→证明级奖励”的两阶段强化学习，最后辅以测试时扩展。所得 30B-A3B 模型 SU-01 仅用约 34 万条轨迹与 200 步 RL 训练，可稳定进行超 10 万 token 的推理，在 IMO 2025/USAMO 2026 与 IPhO 2024/2025 达到金牌水平，并展现出超越数理领域的泛化能力。

SPIN: Structural LLM Planning via Iterative Navigation for Industrial Tasks ◆ 相关
· 2026-05-15
Industrial LLM agents separate planning from execution, but planners often emit structurally invalid or overlong workflows, causing brittle failures and wasted tool/API cost. SPIN is a planning wrapper combining validated DAG planning (with repair prompting) and prefix-based incremental execution that terminates early once a query is answerable. On AssetOpsBench it cuts executed tasks 1061→623 and tool calls 11.81→6.82 per run while raising the accomplished metric 0.638→0.706.
中文摘要
工业级大模型智能体常将规划与执行分离，但规划器易产生结构非法或冗长的工作流，导致脆弱失败与不必要的工具/API 开销。SPIN 是一个规划封装层，结合带修复提示的有效 DAG 规划与基于前缀的增量执行，在足以回答查询时提前终止。在 AssetOpsBench 上将执行任务数由 1061 降至 623、单次工具调用由 11.81 降至 6.82，同时将完成度指标由 0.638 提升至 0.706。

Self-Distilled Agentic Reinforcement Learning ◆ 相关
· 2026-05-15
RL post-training gives only coarse trajectory-level rewards for LLM agents, while applying On-Policy Self-Distillation (OPSD) to multi-turn agents causes compounding instability. SDAR keeps RL as the backbone and treats OPSD as a gated auxiliary objective, mapping detached token-level signals through a sigmoid gate to strengthen teacher-endorsed tokens and attenuate negative rejections. On Qwen2.5/Qwen3 it improves over GRPO by +9.4% (ALFWorld), +7.0% (Search-QA), +10.2% (WebShop-Acc) and avoids naive GRPO+OPSD instability.
中文摘要
强化学习后训练对大模型智能体仅提供粗粒度的轨迹级奖励，而将在线策略自蒸馏（OPSD）用于多轮智能体会带来累积不稳定。SDAR 以 RL 为主干，将 OPSD 作为门控辅助目标，通过 sigmoid 门控调制脱离梯度的 token 级信号：强化教师认可的正增益 token、柔性抑制负向拒绝。在 Qwen2.5/Qwen3 上相比 GRPO 分别提升 9.4%（ALFWorld）、7.0%（Search-QA）、10.2%（WebShop-Acc），并避免了朴素 GRPO+OPSD 的不稳定。

WildClawBench: A Benchmark for Real-World, Long-Horizon Agent Evaluation ◆ 相关
· 2026-05-15
Most agent benchmarks use synthetic sandboxes, short tasks, and mock APIs, leaving real long-horizon native-runtime capability untested. WildClawBench provides 60 human-authored bilingual multimodal tasks (~8 min, 20+ tool calls each) run in reproducible Docker containers hosting real CLI agent harnesses with real tools, graded by rule checks + state auditing + LLM/VLM judge. Across 19 frontier models the best (Claude Opus 4.7 under OpenClaw) reaches only 62.2%, and harness choice alone shifts a model by up to 18 points.
中文摘要
多数智能体基准依赖合成沙箱、短任务与模拟 API，无法检验真实运行环境下的长程能力。WildClawBench 提供 60 个人工编写的双语多模态任务（每个约 8 分钟、20+ 次工具调用），在可复现的 Docker 容器中运行真实 CLI 智能体框架与真实工具，采用规则检查 + 环境状态审计 + LLM/VLM 评判的混合评分。在 19 个前沿模型中表现最好的 Claude Opus 4.7（OpenClaw）也仅达 62.2%，且仅更换执行框架就可使同一模型波动达 18 分。

STALE: Can LLM Agents Know When Their Memories Are No Longer Valid? ◆ 相关
· 2026-05-15
Current LLM-agent memory benchmarks measure static fact retrieval and overlook belief revision when new evidence implicitly invalidates old memories ("Implicit Conflict"). STALE is a benchmark of 400 expert-validated conflict scenarios (1,200 queries, 150K-token contexts) evaluating state resolution, premise resistance, and implicit policy adaptation. Even frontier LLMs reach only 55.2% accuracy; the authors propose CUPMem (structured state consolidation + propagation-aware search) as a baseline.
中文摘要
现有大模型智能体记忆基准主要考察静态事实检索，忽视了新证据隐式推翻旧记忆时的信念修正（“隐式冲突”）。STALE 构建了 400 个专家校验冲突场景（1200 条查询、上下文达 15 万 token），从状态判定、前提抵御与隐式策略适应三个维度评测。即便前沿模型整体准确率也仅 55.2%；作者提出 CUPMem（结构化状态整合 + 传播感知检索）作为改进基线。

EvolveMem:Self-Evolving Memory Architecture via AutoResearch for LLM Agents ◆ 相关
· 2026-05-15
LLM agent memory systems evolve stored content but freeze scoring, fusion, and answer policies at deployment. EvolveMem co-evolves stored knowledge and retrieval mechanisms via an LLM diagnosis module that reads failure logs, finds root causes, and proposes safeguarded configuration changes — treating retrieval config as an action space. It yields a 25.7% improvement on LoCoMo and 18.9% on MemBench, with configurations transferring positively across benchmarks.
中文摘要
现有大模型智能体记忆系统只更新存储内容，而评分、融合与回答策略在部署后被冻结。EvolveMem 让存储知识与检索机制协同演化：通过 LLM 诊断模块读取失败日志、定位根因并提出带安全护栏的配置调整，将检索配置视为动作空间。该方法在 LoCoMo 上提升 25.7%、MemBench 上提升 18.9%，且配置可跨基准正向迁移。
