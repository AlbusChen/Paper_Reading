Beyond Individual Intelligence: Surveying Collaboration, Failure Attribution, and Self-Evolution in LLM-based Multi-Agent Systems ★ 高度相关
· 2026-05-17
HF精选multi-agent
LLM-based multi-agent systems improve over single agents via structured collaboration, but tighter coordination amplifies error propagation that is hard to diagnose and rarely yields structural self-improvement. This survey unifies fragmented research under a four-stage 'LIFE' progression (Lay capability, Integrate via collaboration, Find faults via attribution, Evolve via self-improvement), formally characterizing the causal dependencies between adjacent stages. It identifies open challenges at stage boundaries and proposes a cross-stage agenda toward closed-loop, self-organizing multi-agent intelligence.
中文摘要
基于 LLM 的多智能体系统通过结构化协作超越单体智能体，但更紧密的协同也放大了难以诊断、且少有结构性自我改进的错误传播问题。本综述以一个四阶段 “LIFE” 框架（奠定能力、协作集成、归因排错、自我演化）统一以往割裂的研究线索，形式化刻画相邻阶段间的因果依赖，指出阶段边界处的开放挑战，并提出面向闭环、自组织多智能体智能的跨阶段研究议程。

Learning to Communicate Locally for Large-Scale Multi-Agent Pathfinding ★ 高度相关
· 2026-05-17
HF精选multi-agent
Learning-based multi-agent pathfinding (MAPF) solvers frame the problem as a per-agent Dec-POMDP solved via RL/IL but typically lack effective inter-agent coordination. This paper proposes LC-MAPF, a generalizable pre-trained model that adds a learnable local communication module enabling multi-round feature exchange between neighboring agents. It outperforms prior IL- and RL-based learning MAPF solvers across diverse unseen scenarios while, unlike most communication-based methods, preserving scalability.
中文摘要
针对学习式多智能体路径规划（MAPF）通常将问题建模为单体 Dec-POMDP 而缺乏有效协同的问题，本文提出 LC-MAPF：一个可泛化的预训练模型，引入可学习的局部通信模块，使相邻智能体进行多轮特征交换以增强协作。在多种未见场景下其性能超过现有基于模仿学习与强化学习的 MAPF 求解器，且不同于多数通信式方法，该通信机制不损害可扩展性。

Achieving Gold-Medal-Level Olympiad Reasoning via Simple and Unified Scaling · 低相关
· 2026-05-17
HF精选Tech Report
This technical report (~77 pages, 28 authors) presents SU-01, a reasoning model reaching gold-medal-level performance on IMO 2025 / USAMO 2026 and strong IPhO 2024/2025 results. It uses a simple unified scaling pipeline: SFT with a reverse-perplexity curriculum to instill proof-search behavior, two-phase RL from verifiable to proof-level rewards, and test-time scaling supporting trajectories beyond 100K tokens. The approach generalizes from competition math/physics to broader scientific reasoning.
中文摘要
本技术报告（约 77 页、28 位作者）提出 SU-01 推理模型，在 IMO 2025/USAMO 2026 达到金牌水平，并在 IPhO 2024/2025 取得优异成绩。其采用简洁统一的扩展流程：以逆困惑度课程进行监督微调以习得证明搜索行为，分两阶段强化学习（从可验证奖励到证明级奖励），并辅以支持超过 10 万 token 轨迹的测试时扩展。该方法可从竞赛数学/物理泛化到更广泛的科学推理。

STALE: Can LLM Agents Know When Their Memories Are No Longer Valid? ◆ 相关
· 2026-05-17
HF精选
LLM-agent memory benchmarks mostly test static fact retrieval, ignoring belief revision when later evidence implicitly invalidates earlier memory. STALE introduces 400 expert-validated implicit-conflict scenarios (1,200 queries, contexts up to 150K tokens) probing State Resolution, Premise Resistance, and Implicit Policy Adaptation. Even the best model reaches only 55.2% accuracy; the proposed CUPMem prototype strengthens write-time state consolidation as a baseline for state-aware memory.
中文摘要
LLM 智能体记忆基准多考察静态事实检索，忽视当后续证据隐式推翻早先记忆时的信念修正。STALE 构建 400 个专家校验的隐式冲突场景（1200 条查询、上下文至多 15 万 token），从状态判定、前提抗性与隐式策略适应三维探测。即便最优模型也仅 55.2% 准确率；所提 CUPMem 原型通过写入时状态整合为状态感知记忆提供基线。

EvolveMem:Self-Evolving Memory Architecture via AutoResearch for LLM Agents ◆ 相关
· 2026-05-17
HF精选
Existing LLM-agent memory systems evolve stored content but keep retrieval scoring/fusion/answer policies frozen. EvolveMem exposes the full retrieval configuration as an action space optimized by an LLM diagnosis module that reads failure logs, infers root causes, and applies guarded adjustments with auto-revert and explore-on-stagnation. This closed-loop 'AutoResearch' yields +25.7% over the strongest baseline on LoCoMo and +18.9% on MemBench, with positive cross-benchmark transfer.
中文摘要

Self-Distilled Agentic Reinforcement Learning ◆ 相关
· 2026-05-17
HF精选
Multi-turn agentic RL gives only coarse trajectory-level rewards, and naively adding token-level self-distillation destabilizes training. SDAR keeps RL as the backbone and treats self-distillation as a gated auxiliary objective, using a sigmoid gate to reinforce endorsed positive-gap tokens while softly attenuating negative teacher signals. It yields consistent gains over GRPO (e.g., +9.4% ALFWorld, +10.2% WebShop, +7.0% Search-QA) across Qwen2.5/Qwen3 scales.
中文摘要
多轮智能体强化学习仅提供粗粒度的轨迹级奖励，简单加入 token 级自蒸馏会破坏训练稳定性。SDAR 以强化学习为主干，将自蒸馏作为受门控的辅助目标，用 sigmoid 门强化被认可的正增益 token，同时柔性衰减负向教师信号。在 Qwen2.5/Qwen3 多种规模上较 GRPO 稳定提升（如 ALFWorld +9.4%、WebShop +10.2%、Search-QA +7.0%）。

WildClawBench: A Benchmark for Real-World, Long-Horizon Agent Evaluation ◆ 相关
· 2026-05-17
HF精选
Most agent benchmarks use synthetic sandboxes, short tasks, and mock APIs, leaving native-runtime long-horizon competence untested. WildClawBench provides 60 human-authored bilingual multimodal tasks (~8 min, 20+ tool calls each) run in reproducible Docker containers with real CLI harnesses (OpenClaw, Claude Code, Codex, Hermes) and hybrid grading. Across 19 frontier models the best (Claude Opus 4.7) reaches only 62.2%, and harness choice alone shifts a model by up to 18 points.

PREPING: Building Agent Memory without Tasks ◆ 相关
· 2026-05-17
HF精选
Agents face a cold-start gap when entering a new environment with no task-specific experience. PREPING builds procedural memory before any target tasks using only self-generated synthetic practice, via a proposer-solver-validator loop where a structured 'proposer memory' controls feasibility, redundancy, and coverage. On AppWorld/BFCL v3/MCP-Universe it matches strong offline/online playbook methods while cutting deployment cost by 2.2-3.0x versus online memory construction.
中文摘要
智能体进入无任务经验的新环境时面临冷启动难题。PREPING 仅用自生成的合成练习，在观察到目标任务前即构建过程性记忆：通过“提议器—求解器—验证器”闭环，并以结构化“提议器记忆”控制任务可行性、冗余度与覆盖度。在 AppWorld/BFCL v3/MCP-Universe 上其性能比肩强离线/在线 playbook 方法，同时相较在线记忆构建将部署成本降低 2.2–3.0 倍。

SPIN: Structural LLM Planning via Iterative Navigation for Industrial Tasks ◆ 相关
· 2026-05-17
HF精选
Industrial LLM agents that separate planning from execution often produce structurally invalid or overlong workflows, causing brittle failures and avoidable tool/API cost. SPIN is a planning wrapper enforcing a validated DAG contract (with repair prompting) and executing DAG prefixes incrementally, stopping once a prefix suffices. On AssetOpsBench it cuts executed tasks 1061->623 and tool calls 11.81->6.82 per run while improving accomplishment 0.638->0.706, with consistent gains on MCP Bench.
中文摘要
将规划与执行分离的工业级 LLM 智能体常生成结构非法或过长的工作流，导致脆弱失败与不必要的工具/API 开销。SPIN 是一个规划包装器，强制经校验的 DAG 契约（含修复提示），并按前缀增量执行，一旦前缀足以回答即停止。在 AssetOpsBench 上将执行任务由 1061 降至 623、单次工具调用由 11.81 降至 6.82，同时完成度由 0.638 提升至 0.706，并在 MCP Bench 上稳定提升。
