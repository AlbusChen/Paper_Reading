When Cloud Agents Meet Device Agents: Lessons from Hybrid Multi-Agent Systems ★ 高度相关
· 2026-05-31
multi-agent
Hybrid multi-agent systems pair cloud-hosted frontier LLMs with cost-efficient on-device SLMs, but their design space—which tightly couples task accuracy, monetary cost, and edge energy—is poorly understood and usually handled ad hoc. The authors adapt two representative MAS architectures to support hybrid inference and map how each design choice shifts the operating point along the power/cost/performance Pareto frontier. They find the optimal architecture is highly task-dependent and that more frontier-level compute does not reliably translate to better performance, though SLMs can benefit substantially from selective LLM assistance.
中文摘要
本文系统研究了云端大模型(LLM)与端侧小模型(SLM)协同的混合多智能体系统设计空间，该空间将任务精度、金钱成本与边缘能耗紧密耦合，却长期缺乏通用设计原则。作者将两种代表性多智能体架构改造为支持混合推理，刻画了各项设计选择如何在“功耗—成本—性能”帕累托前沿上移动工作点。实验表明最优架构高度依赖具体任务，单纯增加云端算力并不能稳定提升性能，但小模型能从有选择的大模型协助中显著获益。

Towards Verifiable Multimodal Deep Research: A Multi-Agent Harness for Interleaved Report Generation ★ 高度相关
· 2026-05-31
multi-agent
Verifiable multimodal "deep research" is hard because long-form synthesis lacks deterministic ground truth and must interleave textual arguments with visual evidence. PTAH is a multi-agent harness that orchestrates planning, research, and writing stages: specialized agents build visual-aware plans, gather claim-grounded evidence, maintain source-aligned images in a Visual Working Memory, and compose reports via declarative multimodal tool use, while a verifier agent acts as the acceptance function enforcing factual grounding, citation fidelity, and cross-modal consistency. With its companion PTAHEval protocol, it produces more reliable and usable multimodal reports than strong baselines.
中文摘要
可验证的多模态“深度研究”颇具挑战：长篇综合缺乏确定性标准答案，且需将文字论证与视觉证据交织呈现。本文提出多智能体框架 PTAH，统筹规划、检索与写作三个阶段：由专职智能体构建视觉感知的计划、收集可溯源的证据、在“视觉工作记忆”中维护与来源对齐的图像，并通过声明式多模态工具调用生成报告；同时设置验证器智能体作为验收函数，贯穿全流程地强制事实依据、引用一致性与跨模态一致性。配合提出的 PTAHEval 评测协议，其生成的多模态报告比强基线更可靠、更具可读性。

AsyncTool: Evaluating the Asynchronous Function Calling Capability under Multi-Task Scenarios ★ 高度相关
· 2026-05-31
Existing tool-use benchmarks ignore the temporal dimension—tool response latency and concurrent multi-tasking—so they miss whether an agent can exploit idle time while waiting on tools (asynchronous tool calling). AsyncTool presents multiple heterogeneous tasks simultaneously with simulated realistic latency, built via a hybrid data-evolution strategy, and scores agents at step, sub-task, and task levels with efficiency-oriented coordination metrics. Experiments show delayed feedback substantially degrades current agents, and models that better coordinate task switching, dependency tracking, and state maintenance perform best.
中文摘要
现有工具调用评测大多忽视时间维度，既不考虑工具响应延迟，也局限于单任务，因而无法衡量智能体能否在等待工具返回时利用空闲时间（即异步工具调用能力）。本文提出 AsyncTool 基准，在交互式多任务环境中同时给出多个异构任务并模拟真实的工具延迟，采用混合数据演化策略构建数据集，并从步骤、子任务、任务三个层面引入面向效率的协调度量。实验显示延迟反馈会明显拉低现有智能体表现，而更擅长任务切换、依赖追踪与状态维护的模型协调效率更高。

PANDO: Efficient Multimodal AI Agents via Online Skill Distillation ★ 高度相关
· 2026-05-31
Multimodal web agents increasingly buy performance with extra inference-time compute (rollout search, verifier passes, skill discovery), raising whether an agent can instead grow more efficient with experience. PANDO is a single-rollout online skill-distillation framework with a structured Skill Library, progress reflection, confidence-based skill demotion, hierarchical routing, visual compression, and cache-aware prompting. On all 910 VisualWebArena tasks it reaches 58.3% success while using 58–61% fewer tokens than SGV/WALT baselines, and it introduces three trajectory-level efficiency metrics (action-repetition rate, step-overhead ratio, prompt-cache utilization).
中文摘要
多模态网页智能体往往依靠增加推理期算力（回放搜索、验证器、离线技能发现等）来换取性能提升，本文转而追问：智能体能否随经验积累而变得更高效而非更昂贵？作者从 VisualWebArena 轨迹中归纳出重复动作循环、隐性探索开销、提示缓存复用低三类低效来源，提出单次回放的在线技能蒸馏框架 PANDO，融合结构化技能库、进度反思、置信度降级、分层路由、视觉压缩与缓存感知提示。在全部 910 个任务上取得 58.3% 成功率，较 SGV/WALT 基线减少 58%–61% 的 token，并提出三项轨迹级效率指标使效率可被显式度量。

Discovering Cooperative Pipelines: Autoresearch for Sequential Social Dilemmas ★ 高度相关
· 2026-05-31
The paper studies two-level autoresearch for cooperation: an outer-loop coding agent autonomously redesigns the inner-loop pipeline (system prompts, feedback functions, helper libraries, iteration logic) of an LLM policy-synthesis system for multi-agent Sequential Social Dilemmas. Across two games, two synthesizer LLMs, and two welfare objectives, the researcher agent reliably beats hand-designed baselines, sharply reduces run-to-run variance, and outperforms prompt-only optimization. Notably the discovered pipelines are objective-dependent—an explicit fairness mechanism emerges only under the Rawlsian maximin objective—supporting an information-design reading of what the researcher reveals to a boundedly rational synthesizer.
中文摘要
本文研究面向合作的两层“自动研究”框架：外层编码智能体自主重写内层 LLM 策略合成系统的流水线（系统提示、反馈函数、辅助库与迭代逻辑），以求解多智能体序贯社会困境(SSD)。在两个博弈、两个合成器模型与两种社会福利目标下，研究者智能体稳定超越人工设计基线，大幅收紧运行间方差，并优于仅做提示优化的方法。值得注意的是所发现的流水线依目标而异——只有在 Rawls 最小最大(maximin)目标下才会注入显式公平机制，这支持了一种“信息设计”解读：研究者会依据福利目标决定向有限理性的合成器揭示哪些信息。

LiteCoder-Terminal: Scaling Long-Horizon Terminal Environments for Learning Language Agents ◆ 相关
· 2026-05-31
Training terminal-mastering language agents is bottlenecked by reliance on scraped external repositories, limiting domain diversity, environment controllability, and targeted capability training. LiteCoder-Terminal-Gen is a zero-dependency synthesis pipeline that autonomously generates executable, verifiable terminal environments from domain specs, yielding an 11,255-trajectory SFT set and 602 verifiable RL environments. SFT on Qwen models markedly beats base models (a 32B variant reaches 34.0% pass@1 on Terminal Bench Pro), and multi-turn preference optimization adds further gains—showing fully synthetic executable environments give scalable, verifiable supervision.
中文摘要
训练能驾驭终端环境的语言智能体长期受制于对外部抓取代码库的依赖，导致领域多样性、环境可控性与针对性能力训练均受限。本文提出零依赖合成流水线 LiteCoder-Terminal-Gen，可直接依据领域规格自主生成可执行、可验证的终端训练环境，并构建了含 11,255 条专家轨迹的 SFT 数据集与 602 个可验证 RL 环境。在 Qwen 系列上微调显著超越基座模型（32B 变体在 Terminal Bench Pro 上达 34.0% pass@1），叠加多轮偏好优化进一步提升，表明全合成可执行环境能提供可扩展且可验证的监督信号。

Skill0.5: Joint Skill Internalization and Utilization for Out-of-Distribution Generalization in Agentic Reinforcement Learning ◆ 相关
· 2026-05-31
Skill-based RL for agents forces a rigid choice between full skill externalization (heavy context overhead) and full internalization (overfitting and knowledge conflicts). Skill0.5 instead differentiates treatment via a difficulty-aware router: it internalizes general skills through privileged distillation to build a cognitive foundation for hard tasks, while using diagnostic probing on easy tasks to penalize shortcuts and enforce task-specific skill utilization. On ALFWorld and WebShop it beats memory- and skill-based RL baselines in- and out-of-distribution while cutting the context overhead of full externalization.
中文摘要
面向智能体的技能型强化学习通常被迫在“完全外置技能（上下文开销过大）”与“完全内化技能（易过拟合、知识冲突）”之间二选一。本文提出 Skill0.5，借助难度感知路由器区别对待两类技能：对困难任务通过特权蒸馏内化通用技能以建立认知基础，对简单任务用诊断式探查惩罚走捷径并强制使用任务专用技能。在 ALFWorld 与 WebShop 上，该方法在分布内外均优于基于记忆与基于技能的强化学习基线，同时避免了完全外置带来的上下文开销。

UI-KOBE: Knowledge-Oriented Behavior Exploration for Lightweight Graph-Guided GUI Agents ◆ 相关
· 2026-05-31
Effective mobile GUI agents usually rely on large vision-language models, but lightweight on-device agents (cheaper, more private) remain unreliable at end-to-end planning from screenshots alone. UI-KOBE first autonomously explores an app to build an app knowledge graph (nodes = UI states, edges = executable transitions), then at runtime lets a lightweight agent localize the current node and choose among self-loop actions, neighboring transitions, completion, or fallback free actions. This graph guidance reduces the end-to-end planning burden, making small models more effective, interpretable, and privacy-preserving on-device.
中文摘要
高效的移动端 GUI 智能体通常依赖大型视觉语言模型，而更廉价、更利于隐私的端侧轻量智能体在仅凭截图端到端规划时仍不可靠。本文提出 UI-KOBE：先自主探索应用并构建“应用知识图”（节点为界面状态，边为可执行跳转），运行时让轻量智能体定位当前节点，并在自环动作、邻接跳转、任务完成或兜底自由动作之间选择。借助应用级图谱引导，该方法降低了端到端规划负担，使小模型在端侧更有效、更可解释、更注重隐私。

REPOT: Recoverable Program-of-Thought via Checkpoint Repair ◆ 相关
· 2026-05-31
One-shot Program-of-Thought emits a Python plan whose single invalid action silently invalidates the whole trajectory. RePoT adds a deterministic verified replay that walks the plan to its first invalid transition, then makes one LLM call to resume from the verified prefix—costing at most one extra call on the ~14% of failing problems. It beats PoT by +3 to +11 pp across four closed models (peaking 96.9% vs 86.3% on gpt-5.4-mini) and shows that checkpoint information, not the specific prefix tail, is the load-bearing recovery signal, with an Adaptive RePoT dispatcher routing between suffix repair and a fresh retry.
中文摘要
一次性的思维程序(PoT)会输出一段 Python 计划，其中任一非法动作都会悄无声息地令整条轨迹失效。本文提出 RePoT：先用确定性的“已验证回放”把计划执行到首个非法跳转处，再以一次 LLM 调用从已验证前缀续写恢复，仅在约 14% 失败问题上至多多花一次调用。在四种闭源模型上较 PoT 提升 3–11 个百分点（在 gpt-5.4-mini 上 96.9% vs 86.3%），并表明真正起作用的恢复信号是“检查点信息”本身而非具体前缀尾部；作者进一步提出依据已验证前缀长度在后缀修复与重试间路由的 Adaptive RePoT。

CoHyDE: Iterative Co-Training of LLM Rewriter & Dense Encoder for Tool Retrieval ◆ 相关
· 2026-05-31
Tool retrieval over large API catalogs bottlenecks LLM agents because colloquial, underspecified queries don't match technical API vocabulary, and the two dominant approaches (contrastive encoder fine-tuning vs. frozen-LLM HyDE query expansion) fail in complementary ways. CoHyDE co-trains the dense encoder and LLM rewriter as one co-evolving system: the encoder is retrained with InfoNCE on catalog-style hypothetical descriptions from the rewriter, and the rewriter is DPO-aligned against the encoder's retrieval scores. On a ~10k-tool ToolBench subset, three rounds improve NDCG@5 by +2.5 pp on standard and +6.3 pp on vague queries (up to +8 pp on the hardest tier), with co-training being the key ingredient.
中文摘要
在大规模 API 目录上做工具检索是 LLM 智能体的关键瓶颈：用户查询口语化且常欠规范，而目录使用技术性 API 术语，主流的两种方法（对比式编码器微调、冻结 LLM 的 HyDE 查询扩展）在互补的方向上各自失效。本文提出 CoHyDE，将稠密编码器与 LLM 改写器作为一个协同进化系统联合训练：编码器用 InfoNCE 在改写器产生的“目录风格假设描述”上重训，改写器则依据编码器的检索得分用 DPO 进行偏好对齐。在约 1 万工具的 ToolBench 子集上，三轮迭代使标准查询 NDCG@5 提升 2.5 个百分点、模糊查询提升 6.3 个百分点（最难档位最高 +8），消融实验证明协同训练是关键。

PhoneWorld: Scaling Phone-Use Agent Environments ◆ 相关
· 2026-05-31
A core bottleneck for phone-use agents is that controllable, reproducible environments covering real mobile behavior are hard to build at scale. PhoneWorld is a reusable pipeline that converts real GUI trajectories and screenshots into controllable mock-Android environments, executable tasks, automatic verifiers, and training rollouts, currently spanning 34 apps across 16 domains. Under a fixed training budget, swapping 10K AndroidWorld steps for broad PhoneWorld supervision improves four benchmarks at once (e.g., +14.7 on AndroidWorld), and expanding app coverage yields even larger gains—shifting the focus from one benchmark at a time to scaling the supply of environments.
中文摘要
手机操作智能体的核心瓶颈在于：覆盖真实移动行为、可控且可复现的环境难以大规模构建。本文提出 PhoneWorld，一条可复用的流水线，将真实 GUI 轨迹与截图转化为可控的模拟安卓环境、可执行任务、自动验证器与训练回放，目前覆盖 16 个领域的 34 款应用。在固定训练预算下，用 PhoneWorld 的广泛监督替换 1 万步 AndroidWorld 数据可同时提升四个基准（如 AndroidWorld +14.7 分），且扩大应用覆盖带来的收益更大——研究重心由“逐个构建基准”转向“规模化扩充环境供给”。
